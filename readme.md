## rename.js 参数参考手册

Sub-Store 脚本操作中引用此脚本，URL 传参格式：

```
https://raw.githubusercontent.com/Keywos/rule/main/rename.js#flag&bl=true&blkey=GPT>ChatGPT+NF
```

- `#` 后紧跟第一个参数
- 多个参数用 `&` 连接
- 纯开关参数直接写 key，带值的参数用 `key=value`

### 一、地区识别

#### `in` — 输入识别模式

| 传值 | 别名 | 含义 |
|---|---|---|
| 不传 | auto | 自动依次尝试：中文 → 国旗 → 英文全称 → 英文缩写 |
| `zh` / `cn` | — | 只识别中文，如"新加坡""美国" |
| `en` / `us` | — | 只识别英文缩写 |
| `flag` / `gq` | — | 只识别国旗 emoji |
| `quan` | — | 只识别英文全称 |

不传时依次查找中文表 → 国旗表 → 英文全称表 → 英文缩写表，命中即停。

#### `out` — 输出名称格式

| 传值 | 别名 | 含义 |
|---|---|---|
| 不传 | — | 默认输出中文地区名 |
| `zh` / `cn` | — | 输出中文 |
| `en` / `us` | — | 输出英文缩写 |
| `flag` / `gq` | — | 输出国旗 emoji |
| `quan` | — | 输出英文全称 |

示例：`in=zh&out=us` → 输入识别中文，输出转为 `SG`

#### `flag` — 节点名加国旗

开关参数。匹配到地区后，在名前加上对应国旗 emoji。台湾 flag 自动转为 🇨🇳。

```
输入: 新加坡 01
输出: 🇸🇬 新加坡 01
```
#### `nm` — 保留未匹配节点

开关参数。默认情况下，无法识别地区的节点会被丢弃。开启后保留原样（或加 `name=` 前缀）。

#### `fgf` — 前缀/区域/保留词之间的分隔符

默认空格。例如 `fgf=_` → `新加坡_直连_0.3×_01`

#### `sn` — 区域与序号之间的分隔符

默认空格。例如 `sn=-` → `新加坡-01`

### 二、机场前缀

#### `name` — 机场名称前缀

字符串值，附加到节点名前。

```
name=搬瓦工 → 搬瓦工 新加坡 01
```

#### `nf` — 前缀前置

开关参数。默认 `name=` 加在地区名之后、保留词之前；`nf` 使其置于最前面。

| 模式 | 输出 |
|---|---|
| `name=搬瓦工` | 🇸🇬 搬瓦工 新加坡 01 |
| `name=搬瓦工&nf` | 搬瓦工 🇸🇬 新加坡 01 |

---

### 三、保留机制（核心功能）

**执行顺序**：

```
原始名 → [BUILTIN_RETAIN 默认保留] → [blkey 自定义保留] → [bl 倍率] → [blgd 固定格式保留]
```

#### `BUILTIN_RETAIN` — 内置默认保留词（始终生效）

节点原始名中包含以下关键词即自动保留：

`直连` `专线` `中转` `家宽` `IPLC` `IEPL` `Pro`
自动与 `blkey` / `blgd` 去重。

```
输入: 新加坡 | 12 | 直连 *0.3
输出: 新加坡 直连 0.3× 01
```

#### `blkey` — 自定义保留关键词

字符串，多个用 `+` 连接。区分大小写。

```
blkey=GPT+NF+IPLC
```

*高级用法 — 重命名*：用 `>` 替换。
将节点名中的 `GPT` 替换为 `ChatGPT`，同时保留 `NF`：

```
blkey=GPT>ChatGPT+NF
```

#### `bl` — 正则匹配倍率

开关参数。用正则捕获节点名中的倍率信息并标准化输出。

支持格式：

| 源格式 | 示例 | 输出 |
|---|---|---|
| 后缀 `x` / `×` / `倍` | `0.3x` | `0.3×` |
| 前缀 `x` / `×` / `*` | `*0.3` | `0.3×` |
| 中文"倍率"后数字 | `倍率 0.3` | `0.3×` |

倍率为 1 时不显示（视为普通节点）。

#### `blgd` — 固定格式保留

开关参数。从预置列表中匹配并保留标准化标识：

| 输入关键词 | 输出名称 |
|---|---|
| ˣ² ~ ˣ⁵⁰ | 2× ~ 50× |
| IPLC | IPLC |
| IEPL | IEPL |
| 核心 | Kern |
| 边缘 | Edge |
| 高级 / Pro | Pro |
| 标准 / Std | Std |
| 实验 / Exp | Exp |
| 商宽 / Biz | Biz |
| 家宽 / Fam | Fam |
| 游戏 / game | Game |
| 购物 / Buy | Buy |
| 专线 / Zx | Zx |
| LB | LB |
| cloudflare | CF |
| udp / UDP | UDP |
| gpt / GPT | GPT |
| UDPN | UDPN |

注意：`blgd` **只保留最后一个匹配**。需要多词同时保留请用 `blkey` 或 `BUILTIN_RETAIN`。

### 四、节点过滤

#### `clear` — 清理乱名

开关参数。删除节点名包含以下垃圾关键词的节点：

`套餐` `到期` `有效` `剩余` `版本` `已用` `过期` `失联` `测试` `官方` `网址` `备用` `群` `客服` `网站` `获取` `订阅` `流量` `机场` `下次` `官址` `联系` `邮箱` `工单` `学术` `USE` `USED` `TOTAL` `EXPIRE` `EMAIL`

#### `nx` — 只保留 1 倍率节点

开关参数。过滤掉包含非 1 倍率标识的节点。

#### `blnx` — 只保留高倍率

开关参数。`nx` 的反面——只保留倍率 ≥ 2 的节点。

#### `key` — 仅保留主要地区节点

开关参数。两阶段过滤：先保留含主要地区关键词且含数字的节点，再剔除非正常编号的节点。

---

### 五、排序

#### `blpx` — 按保留标识类型排序

开关参数。**依赖 `bl` 参数**。对已匹配到倍率/特殊标识的节点按其类型分组排列。单独使用无效果。

---

### 六、其他

#### `skip` — 跳过无需处理的节点

字符串，多个用 `+` 连接。包含任一关键词的节点完全跳过重命名处理（保留原名不变）。

```
skip=测试+保留+官方
```

优先级最高，在地区识别和保留机制之前执行。

#### `one` — 单节点去编号

开关参数。如果一个地区只有一个节点，去掉 `01` 后缀。

```
新加坡 01 → 新加坡
```

#### `blockquic` — QUIC 协议控制

| 传值 | 效果 |
|---|---|
| `on` | 阻止 QUIC |
| `off` | 不阻止 QUIC |
| 不传 | 不处理 QUIC 属性 |

#### `debug` — 调试模式

开关参数。启用时额外输出调试信息到控制台。

---

### 七、参数联动与执行顺序

```
原始节点名
    │
    ├── [skip] 命中 → 保留原名，跳过后续所有处理
    │
    ├── [clear|nx|blnx|key] ← 过滤阶段，不符合的节点直接删除
    │
    ├── rurekey 预处理（别名映射，如 Tokyo → 日本）
    │
    ├── [blockquic] ← 设置 QUIC 属性
    │
    ├── BUILTIN_RETAIN（始终生效）
    ├── [blkey] ← 自定义关键词保留
    ├── [blgd] ← 固定格式保留
    ├── [bl] ← 倍率捕获
    │
    ├── [in] → 匹配地区名
    │
    ├── 名称重建（按以下顺序拼接）
    │   [nf]前置前缀    ← name= + nf
    │   [flag]国旗      ← flag
    │   后置前缀        ← name=
    │   地区名          ← 地区匹配结果
    │   BUILTIN_RETAIN  ← 内置保留词（去重后）
    │   [blkey]         ← 自定义保留词
    │   [bl]            ← 倍率
    │   [blgd]          ← 固定格式保留
    │
    ├── [nm] 保底 ← 无法匹配地区的节点
    │
    ├── jxh() 编号 ← 同地区同名称节点加序号
    │
    ├── [one] ← 单节点去掉编号
    │
    ├── [blpx] ← 按保留标识类型排序
    │
    └── [key] ← 最终二次过滤（仅地区类节点）
```

### 典型配置速查

```
# 基本使用（默认行为 + 内置保留词，无需参数）
新加坡 | 12 | 直连 *0.3 → 新加坡 直连 0.3× 01

# 加国旗 + 加机场前缀 + 保留关键词
flag&name=搬瓦工&blkey=GPT+NF
日本 | 03 | GPT *1.5 → 🇯🇵 搬瓦工 日本 GPT 1.5× 01

# 仅倍率 + 保留关键词 + 去乱名
bl=true&blkey=NF&clear
美国 | 05 | NF *2 → 美国 NF 2× 01

# 自定义输出为英文 + 保留家宽中转
out=en&bl=true
日本 | 03 | 家宽 中转 3× → Japan 家宽 中转 3× 01
```
